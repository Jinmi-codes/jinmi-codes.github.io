Infrastructure Engineer Roadmap · Phase 1 of 5

Linux & Networking
Depth

Your foundation. Every cloud instance, container, and CI runner is a Linux box on a network. This phase turns familiarity into mastery — the kind that makes everything else click.

~5 months ~10 hrs/week Linux admin TCP/IP internals Bash scripting Home lab
The four pillars
🐧
Linux administration
Weeks 1–8 · The operating system you'll live in
You already know Linux casually. Now go structural. Learn how the OS actually works — the boot process, how systemd manages services, how the kernel exposes hardware to user space. Understand permissions at the inode level, not just chmod numbers. Know how to diagnose a system that's slow, full, or broken — from the command line, with no GUI. This is the layer everything else sits on.
Boot process & systemd File system & inodes Process management Users & permissions Storage & LVM Package management Logging & journald SSH hardening
🌐
Networking internals
Weeks 6–14 · Overlaps with Linux — runs concurrently
You have Security+ networking knowledge — now operationalise it. Know what actually happens at the packet level when you type a URL. Be able to subnet fast in your head. Understand how DNS resolution actually works, step by step. Configure a firewall, a static route, a VLAN. Capture and read real traffic with Wireshark or tcpdump. This depth is what separates someone who "knows networking" from someone who can debug it at 2am in production.
OSI model in practice TCP/IP internals Subnetting (fast) DNS deep dive DHCP & NAT VLANs & routing iptables & nftables Packet capture TLS internals
📜
Bash scripting (production-grade)
Weeks 3–12 · Woven through everything
You write Python. Bash is different — it's glue. Infrastructure automation is 80% shell scripts. Learn it properly: error handling, argument parsing, logging, idempotency. Write scripts you'd actually run on a server. The goal isn't to replace Python — it's to know when bash is the right tool (startup scripts, cron, one-liners in a pipeline) and write it well when it is.
Variables & arrays Conditionals & loops Functions Error handling (set -euo pipefail) Stdin/stdout/stderr Cron & scheduling awk, sed, grep, cut
🖥️
Home lab
Weeks 1–20 · Runs the entire phase
The lab is not optional. Reading and watching will get you 30% of the way. Breaking and fixing real systems gets you the rest. You need a hypervisor running real VMs — Proxmox on an old PC is the gold standard (free, enterprise-grade). Failing that, VirtualBox on your current machine works fine. You'll spin up Ubuntu Server, Debian, and Rocky Linux (RHEL-based) — knowing both Debian and RHEL families is expected at most infra jobs.
Proxmox (recommended) VirtualBox (fallback) Ubuntu Server 24 LTS Rocky Linux 9 pfSense (router VM) Multiple VMs networked
What to avoid
Don't rush toward certs prematurely. The CompTIA Linux+ and LPIC-1 exist, but they're not the goal of this phase. Understanding is. The cert is a byproduct of actually knowing the material deeply. Memorising dumps is worthless here — infra interviews test whether you can reason through real scenarios, not recite acronyms.

Don't skip the lab because it's inconvenient. Every topic in this phase should be practiced in a real environment, not just read about. If you're reading about LVM and haven't actually created, extended, and deleted a logical volume — you don't know LVM yet.
Schedule assumptions: ~2 hours on weekday evenings (Mon–Fri) + one longer session on Saturday (~3–4 hrs). Sunday is rest/review — never skip it, recovery is part of learning. This comes to roughly 13–14 hrs/week. Adjust to your university load — when exams hit, drop to 1 hr/weekday and compensate on breaks. Consistency over intensity.
Module A — Linux foundations (Weeks 1–4)
Week 1 Linux boot, filesystem, and the shell Core navigation ▾
Monday
Boot process: BIOS → GRUB → kernel → initrd → systemd. Watch it happen in your VM. → Linux Journey: Getting Started
Tuesday
Filesystem hierarchy: Every directory in / and what lives there. Mount points, inodes, hard vs symbolic links. → Linux Journey: Filesystem
Wednesday
Shell mastery: Navigation, file ops, stdin/stdout/stderr, pipes, redirection. Practice: 20 shell exercises on OverTheWire: Bandit (levels 0–10)
Thursday
Text tools: grep, awk, sed, cut, sort, uniq, tr, wc, find, xargs — learn one deeply per day this week. → The Linux Command Line book, Ch 19–20
Friday
VIM: Not just basics — splits, macros, search-replace, .vimrc. You'll live in vim remotely. → vimtutor (run it), then vim-adventures.com
Saturday
Lab session: Set up your first Ubuntu Server 24 LTS VM from scratch. No GUI. SSH into it from your host machine. Complete OverTheWire Bandit levels 10–20. Write 3 bash one-liners to automate something tedious on the system.
Sunday
Rest. Optional: skim the week's topics, no new material.
Week 2 Users, permissions & process management System internals ▾
Monday
Users & groups: /etc/passwd, /etc/shadow, /etc/group structure. useradd, usermod, userdel, passwd, id, who, last. Understand UID/GID 0.
Tuesday
Permissions: rwx bits, octal notation, setuid/setgid/sticky bit, umask, ACLs with getfacl/setfacl. Practice: set up a shared directory with group permissions correctly
Wednesday
Processes: ps, top, htop, pstree. fork(), exec(), orphans, zombies. Signals — SIGTERM, SIGKILL, SIGHUP and when to use each. kill, pkill, killall.
Thursday
Job control & scheduling: bg, fg, jobs, nohup, screen/tmux. cron syntax deeply — crontab -e, /etc/cron.d/, /etc/cron.daily/. Write a cron job that logs disk usage daily
Friday
sudo & PAM: /etc/sudoers properly (visudo), sudoers.d/, specific command grants without full root. PAM basics — pluggable auth modules.
Saturday
Lab: Create 3 users with different privilege levels. Set up a shared project directory with correct group permissions. Write a monitoring script that alerts (echo to a log) when a process dies. Practice killing and restarting services manually.
Sunday
Rest.
Week 3 systemd, services & logging Service management ▾
Monday
systemd deep dive: Units, targets, dependencies. systemctl start/stop/enable/disable/status/reload. Understand the difference between restart and reload.
Tuesday
Writing unit files: [Unit], [Service], [Install] sections. ExecStart, Restart=always, After=network.target, EnvironmentFile. Write a unit file for a Python script you've made
Wednesday
journald & logging: journalctl flags — -u, -f, -b, --since, --until, -p. /var/log/ structure. logrotate config. rsyslog basics.
Thursday
Bash scripting — start properly: set -euo pipefail. Functions, local variables, return codes, $?, $#, $@. Argument parsing with getopts. → The Linux Command Line book, Ch 24–27
Friday
Package management: apt vs dnf internals. Repositories, GPG keys, held packages, pinning. rpm vs deb format differences. Building from source (./configure && make && make install).
Saturday
Lab: Set up a second VM (Rocky Linux 9 — your first RHEL-based system). Install and configure nginx. Write a systemd unit file for a custom service. Configure log rotation for it. Practice: break a service and diagnose it using journalctl alone.
Sunday
Rest.
Week 4 Storage, LVM & SSH hardening Storage & remote access ▾
Monday
Storage fundamentals: Block devices, partitions, MBR vs GPT, fdisk/gdisk, /proc/partitions. df, du, lsblk, blkid.
Tuesday
Filesystems: ext4, xfs, btrfs — when to use each. mkfs, mount, /etc/fstab, UUID mounting. tmpfs and ramfs.
Wednesday
LVM (Logical Volume Manager): Physical volumes → volume groups → logical volumes. pvcreate, vgcreate, lvcreate, lvextend, resize2fs. Snapshots. Actually create, extend, and snapshot an LV in your lab
Thursday
SSH hardening: Key-based auth, /etc/ssh/sshd_config — disable root login, disable password auth, change port, AllowUsers. ssh_config on the client. ssh-agent, ssh-add, ProxyJump.
Friday
Bash project day: Write a disk usage alert script — checks all mounted filesystems, logs anything over a threshold, sends a formatted report. Error handling, logging, idempotent. → shellcheck.net to lint your scripts
Saturday
Lab: Add a new virtual disk to your VM. Partition it, format with ext4, mount persistently via fstab. Set up LVM on another disk, create a volume, extend it, take a snapshot. Harden SSH on both VMs — key auth only, no root login. Set up a jump host pattern between your two VMs.
Sunday
Rest. Module A review — write down 5 things you're still fuzzy on and look them up.
Module B — Networking internals (Weeks 5–10)
Week 5 TCP/IP internals & subnetting fluency Packet level ▾
Monday
OSI model operationalised: Not just names — what tool operates at each layer. Where do ping, traceroute, tcpdump, curl, nmap each sit? Map your Security+ knowledge to real Linux commands.
Tuesday
IP addressing deep dive: Binary representation, class A/B/C history (and why it's obsolete), CIDR notation. Private ranges (10.x, 172.16-31.x, 192.168.x). Loopback, APIPA.
Wednesday
Subnetting — become fast: /24 → /30 binary math. Variable-length subnet masking (VLSM). Practice: subnettingpractice.com — do 20 problems a day until you can do /24 splits in under 30 seconds mentally
Thursday
TCP deep dive: 3-way handshake, 4-way termination, sequence numbers, ACKs, sliding window, congestion control (CUBIC, BBR). What TIME_WAIT is and why it exists.
Friday
UDP & ICMP: When UDP beats TCP and why (DNS, streaming, gaming). ICMP types — echo, unreachable, TTL exceeded. How traceroute actually works (TTL trick).
Saturday
Lab — Wireshark/tcpdump: Capture a DNS query, HTTP request, and TCP handshake. Read the raw packets. Filter by protocol, port, IP. Understand every field in an IP header and TCP header. This is the most important Saturday of this module.
Sunday
Rest. Keep doing subnetting practice (10 mins only).
Week 6 DNS internals & DHCP Name resolution ▾
Monday
DNS resolution step by step: Recursive resolver → root servers → TLD → authoritative. What actually happens when you type google.com. Cache TTLs, negative caching.
Tuesday
DNS record types: A, AAAA, CNAME, MX, TXT, NS, SOA, PTR, SRV. When each is used. SPF, DKIM, DMARC records (ties to your Security+ background).
Wednesday
DNS tools: dig extensively — +trace, +short, @resolver, -t MX. nslookup (legacy). host. Read a full dig +trace output and explain every line. Trace 5 different domains, compare resolver paths
Thursday
DHCP mechanics: DORA process (Discover, Offer, Request, Acknowledge). Lease times, reservations by MAC, DHCP relay agents. /var/lib/dhcp/dhclient.leases.
Friday
Bash + networking: Write scripts using curl, wget, dig, ping, nc (netcat). A script that checks if a list of hosts are reachable and logs results with timestamps.
Saturday
Lab: Set up a local DNS server (bind9 or dnsmasq) in a VM. Create A records for your lab VMs so they resolve by hostname. Set up DHCP for your lab network. Test with dig against your own resolver. Capture DNS traffic in Wireshark.
Sunday
Rest.
Week 7 Routing, VLANs & NAT Network architecture ▾
Monday
Routing fundamentals: ip route, routing tables, default gateway, static routes. How a packet decides which interface to leave from. Longest-prefix match.
Tuesday
Dynamic routing overview: OSPF and BGP concepts (you won't configure BGP yet, but know what it does). Why ISPs use BGP. AS numbers. Route redistribution concept.
Wednesday
VLANs: 802.1Q tagging, trunk vs access ports, inter-VLAN routing. How to segment a network into VLANs logically. Why you'd want to isolate a VLAN (security, broadcast domains).
Thursday
NAT & PAT: Source NAT, destination NAT, masquerade. How your home router gives you internet with one public IP. iptables MASQUERADE rule. Port forwarding.
Friday
Linux as a router: ip_forward, iptables NAT rules, setting up a Linux VM as a gateway. iproute2 toolkit — ip addr, ip link, ip route, ip neigh (ARP table).
Saturday
Lab: Set up pfSense VM as your lab router/firewall. Create two separate VM networks (subnets). Route between them through pfSense. Set up NAT so lab VMs can reach the internet through pfSense. Capture a packet leaving one subnet and arriving in another — see the TTL decrement and MAC change.
Sunday
Rest.
Week 8 Firewalls — iptables & nftables Traffic control ▾
Monday
iptables architecture: Tables (filter, nat, mangle, raw), chains (INPUT, OUTPUT, FORWARD, PREROUTING, POSTROUTING), rules, targets (ACCEPT, DROP, REJECT, LOG). Packet flow through all chains.
Tuesday
Writing iptables rules: Allow established/related, drop invalid, allow specific ports, rate limiting with hashlimit, logging dropped packets. iptables-save and iptables-restore. Write a baseline firewall ruleset for a web server
Wednesday
nftables: The modern replacement for iptables. nft syntax, tables, chains, sets, maps. How to translate an iptables ruleset to nftables. ufw as a front-end (and what it does behind the scenes).
Thursday
TLS/SSL internals: Handshake steps (Client Hello → Server Hello → Certificate → Key Exchange → Finished). Cipher suites, certificate chains, SNI, OCSP. How Let's Encrypt works. openssl s_client to inspect a TLS connection.
Friday
Bash — networking scripts: Port scanner (using /dev/tcp or nc), service checker, SSL cert expiry checker. Write a script that checks if a cert expires within 30 days and alerts
Saturday
Lab: Apply a proper iptables ruleset to your Ubuntu VM — default DROP, allow SSH, HTTP, HTTPS, established. Test each rule. Break one rule intentionally and diagnose why traffic is being dropped using iptables -v -n -L and tcpdump simultaneously. Repeat with nftables.
Sunday
Rest.
Module C — Bash scripting projects (Weeks 9–14, concurrent)
Weeks 9–10 Advanced bash + system performance Scripting & tuning ▾
Week 9
System performance: CPU — load average, context switches, iostat, mpstat. Memory — free, vmstat, /proc/meminfo, swap usage. I/O — iotop, iostat, iowait. What these numbers mean and when to be alarmed. → Brendan Gregg's Linux Performance tools poster (free, legendary)
Week 10
Advanced bash: Here-docs, process substitution, coprocesses. Bash arrays (indexed and associative). String manipulation without external tools. Trap signals in scripts for cleanup. Parallel execution with & and wait. Build a sysinfo report script that outputs structured data for all metrics learned so far
Module D — Integration & capstone (Weeks 11–20)
Weeks 11–14 Security hardening, NFS/Samba & automation Production patterns ▾
Week 11
Linux hardening: CIS Benchmark for Ubuntu/Rocky. SELinux (on Rocky) and AppArmor (on Ubuntu) — modes, profiles, audit2allow. Fail2ban config. sysctl security parameters (/etc/sysctl.conf). auditd for system call auditing.
Week 12
Network file systems: NFS v4 — exports, mount options, security (Kerberos optional). Samba — smb.conf, shares, auth. AutoFS for automounting. Lab: set up shared NFS storage between your VMs, used by a service on both
Week 13
Network monitoring: nmap for host discovery and port scanning. netstat/ss — understand every column. Bandwidth monitoring: iftop, nethogs, nload. Detecting suspicious connections.
Week 14
Bash capstone project: Write a full server health monitoring script. Checks CPU, memory, disk, zombie processes, failed systemd services, open ports vs expected, SSL cert expiry. Outputs a structured report. Runs via cron. Sends alert to a log file (and optionally email via mailx). → This becomes a portfolio piece
Weeks 15–20 Capstone lab build & phase review Portfolio + readiness ▾
Weeks 15–17
Full lab environment build: Multi-VM network with pfSense router, Ubuntu web server, Rocky Linux app server, NFS storage VM. All hardened, all monitored by your scripts, all SSH key-only. Documented with a network diagram and runbook. This is your Phase 1 capstone.
Weeks 18–19
Deep review + gap filling: Go back to anything you felt shaky on. Do the OverTheWire: Natas challenges (web/networking focused). Re-read your notes. Explain concepts out loud as if teaching someone else.
Week 20
Optional: CompTIA Linux+: If you've done the work, this cert now takes 2–3 weeks of focused prep (not 3 months). Use Professor Messer and Jason Dion practice exams. You'll pass comfortably. This is a nice credential but not the goal — the skills are. Then: begin Phase 2.
Progressive lab projects

These build on each other. Each one is a real thing you've built that you can talk about in an interview. Document everything in a GitHub repo with a README — this becomes your portfolio.

Project 01 · Week 1–2
Hardened SSH jump host
Set up an Ubuntu Server VM accessible only via SSH key auth. Disable root login, disable password auth, change the default port, configure AllowUsers. Set up a second VM that's only accessible by jumping through the first. Write a bash script that automates the hardening steps idempotently — run it twice and the second run makes no changes.
SSH config mastery Idempotent scripting ProxyJump pattern Portfolio: GitHub README
Project 02 · Week 3–4
Self-healing web service
Deploy nginx as a systemd service on Rocky Linux. Write a custom systemd unit file with Restart=always and a startup delay. Add a watchdog script (run by a timer unit, not cron) that checks if nginx responds to an HTTP request — if it fails 3 times, it logs the failure and restarts the service. Set up log rotation for nginx and your watchdog logs.
systemd units & timers Service health checking Log rotation Rocky Linux (RHEL family)
Project 03 · Week 5–8
Lab network with pfSense
Build a proper internal lab network. pfSense VM acts as the router and DHCP server. Two isolated subnets: one for "web" VMs, one for "db" VMs. Set up firewall rules so web VMs can only talk to db VMs on port 5432. DB VMs cannot initiate connections to web VMs. Set up your local DNS so all VMs resolve by hostname. Capture and document the traffic flow with Wireshark.
pfSense router/firewall Multi-subnet design Firewall rules (real use case) Local DNS Wireshark analysis
Project 04 · Week 9–14
Server health monitoring system
A complete bash monitoring suite. Runs via cron on every VM in your lab. Checks: CPU load (alert if >80% sustained), memory usage, disk usage per mount point, zombie processes, failed systemd services, unexpected open ports (compares current ss output against a known-good baseline), SSL cert expiry for any HTTPS services. Outputs a structured daily report. Aggregates reports from all VMs via NFS shared storage. One master script you run to see the health of your entire lab.
Production bash scripting Multi-host awareness NFS integration Baseline deviation detection Portfolio centrepiece

You're ready for Phase 2 when you can confidently do all of these from memory, in a live terminal, without googling. Be honest with yourself — these are the real bar.

Linux
  • Explain the Linux boot process from power-on to login prompt, naming each stage and what could go wrong at each
  • Diagnose a slow or unresponsive server using only top, iostat, vmstat, ss, journalctl — and name the most likely cause from the output
  • Set up a new user with specific sudo privileges (only certain commands), correct primary group, home directory, and SSH key — without a GUI
  • Create and extend an LVM logical volume on a new disk, mount it persistently, and take a snapshot — all from scratch
  • Write a systemd unit file for a custom service with proper restart policy, environment file, and dependency ordering
  • Diagnose why a service failed using journalctl alone, fix it, and prevent the issue recurring
  • Write a production bash script with proper error handling (set -euo pipefail), logging, argument parsing, and idempotency
  • Harden a fresh Linux server — SSH key-only, no root login, firewall baseline, fail2ban, disable unnecessary services — without a guide
Networking
  • Subnet any /24 network into any prefix length (/25 through /30) in under 60 seconds, mentally
  • Trace a DNS query step by step with dig +trace and explain every line of output
  • Capture and read a TCP 3-way handshake in Wireshark/tcpdump and explain each packet's fields
  • Write an iptables ruleset for a web server — default DROP, allow SSH, HTTP, HTTPS, established, reject all else — and test it
  • Set up routing between two subnets using a Linux VM as the router, with correct iptables NAT rules
  • Explain TLS handshake steps and inspect a real connection with openssl s_client
  • Diagnose a network connectivity issue using ping, traceroute, ss, tcpdump, dig — and know which tool to reach for first at each layer
Lab
  • Your lab runs all 4 projects — jump host, self-healing service, pfSense network, monitoring system — and they work
  • Your GitHub repo is documented — each project has a README explaining what it does, how to set it up, and what you learned
  • You've broken and recovered from at least: a corrupted fstab, a misconfigured sshd_config that locked you out, a firewall rule that dropped all traffic, a failed service that wouldn't restart
